# DIAGRAM_TEMPLATES.md

本文件提供 Codex 可直接套用的常用模板。  
每個模板分成三部分：

1. 使用情境
2. 結構描述建議
3. 簡報輸出規則

---

## 模板 1：審查流程圖

### 使用情境
- concept review flow
- 申請與審查流程
- 教案審查流程
- 專案核准流程

### 結構描述
- 使用 Mermaid flowchart
- 方向 `LR`
- 決策點不宜超過 3 個
- 支線不可過多

### 簡報輸出規則
- 標題置頂置中
- 主流程節點用深藍
- 決策節點用橘色
- 補充說明框用淺灰
- 每列最多 4 個節點
- 輔助流程使用虛線
- 圖例置於右下角

## 模板 2：平台架構圖

### 使用情境
- Odoo + Vue + API + DB
- 醫院能源戰情平台
- AI 平台整體架構
- 雲邊整合平台

### 結構描述
- 使用 PlantUML component 或 Mermaid architecture
- 依層分區：
  - 使用者層
  - 應用層
  - 服務層
  - 資料層
  - 外部整合層

### 簡報輸出規則
- 採上下分層
- 每層最多 4 個核心模組
- 同層模組同寬同高
- 前端層藍色
- 後端層藍綠色
- 資料層灰色
- 外部整合層橘色註記
- 箭頭儘量由上往下或左往右

## 模板 3：網路拓樸圖

### 使用情境
- PVE / Ceph / LXC / VM
- WireGuard / 內外網節點
- IoT Gateway / InfluxDB / Dashboard 節點關係

### 結構描述

- 核心節點置中

- 線條不可過度交叉

## 模板 4：提案型比較圖

### 使用情境
- 方案 A / B 比較

- 校內版 vs 產業版

- 左右雙欄或三欄
- 每欄使用固定色彩
- 相同面向在同一列
- 表格、色塊、圖示可混搭

## 模板 7：圓餅圖 / 環圈圖

### 使用情境
- 市占率
- 預算結構
- 類別占比
- 問卷選項比例

### 適用條件
- 類別數 2~5 個最佳
- 強調比例結構，不強調精確比較
- 若類別太多或差異太小，改用長條圖

### 資料格式建議
{
  "title": "預算結構占比",
  "chart_type": "pie",
  "labels": ["設備", "人事", "材料", "其他"],
  "values": [45, 30, 15, 10],
  "unit": "%"
}

### 簡報輸出規則

- 使用環圈圖優先於傳統圓餅圖
- 每色代表固定群組
- 類別名稱與百分比可標示於外側
- 不可使用 3D 圓餅圖
- 若有總額，置於圖中央或副標說明

## 模板 8：長條圖 / 柱狀圖

### 使用情境
- 各組比較
- 課程滿意度比較
- 平台功能得分比較
- KPI 達成率比較

### 適用條件

- 要比較大小時優先使用
- 類別較多時改用水平長條圖
- 資料格式建議
{
  "title": "各平台滿意度比較",
  "chart_type": "bar",
  "categories": ["平台A", "平台B", "平台C"],
  "series": [
    {"name": "滿意度", "values": [4.2, 3.8, 4.5]}
  ],
  "unit": "分"
}

### 簡報輸出規則

- 同系列使用同色
- 若有多系列，使用固定語義色
- 類別名稱過長時用水平長條圖
- 數值可標在柱末端
- 優先由高到低排序

## 模板 9：折線圖 / 趨勢圖

### 使用情境
- 月份趨勢
- KPI 時間變化
- 成長率變化
- 設備耗能趨勢

### 資料格式建議
{
  "title": "月度耗能趨勢",
  "chart_type": "line",
  "categories": ["1月", "2月", "3月", "4月"],
  "series": [
    {"name": "耗能", "values": [120, 115, 130, 110]}
  ],
  "unit": "kWh"
}

### 簡報輸出規則
- 線條不超過 3 條
- 重要系列用主色
- 比較系列用輔色或中性藍綠
- 避免過多資料點標籤
- 異常點或里程碑可額外註記


## 模板 10：堆疊長條圖 / Likert 問卷圖

### 使用情境
- 五點量表
- 滿意度結構
- 同意 / 普通 / 不同意分布
- 類別內部構成比較

### 資料格式建議
{
  "title": "使用者滿意度分布",
  "chart_type": "stacked_bar",
  "categories": ["平台A", "平台B"],
  "series": [
    {"name": "非常同意", "values": [25, 30]},
    {"name": "同意", "values": [40, 35]},
    {"name": "普通", "values": [20, 20]},
    {"name": "不同意", "values": [10, 10]},
    {"name": "非常不同意", "values": [5, 5]}
  ],
  "unit": "%"
}

### 簡報輸出規則
- 色階需有邏輯，正向到負向逐步變化
- 若為 Likert，正向與負向可左右對稱
- 圖例固定右下角
- 適合問卷報告與評鑑簡報

## 模板 11：水平長條圖 / 排名圖

### 使用情境
- 排名前十名
- 重要性排序
- 問項平均數排序
- 問題發生次數排序

### 簡報輸出規則
- 由高到低排序
- 類別名稱長時優先水平長條圖
- 僅保留必要格線
- 前三名可使用強調色
